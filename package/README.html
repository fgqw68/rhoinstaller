RhoElements release notes:

Build:
rhodes-3.3.3.beta.5.2.gem
rhoelements-2.0.0.beta.5.2.gem

Bug fixes:
1. Fix several Windows Mobile and CE issues

New features:

1. Android ET1: Integration with RhoElements v1: support all Plug-ins All Plug-in functionality available throw Ruby, Meta tags and Javascript Support RhoElements v1(Hybrid) applications

2. Windows Mobile and Windows CE: Integration with RhoElements v1: support all Plug-ins All Plug-in functionality available throw Ruby, Meta tags and Javascript

3. Shared runtime for Windows Mobile and CE http://edgedocs.rhomobile.com/rhoelements/moto-api-beta4#rhoelements-shared-runtime It support RhoElements v1 and Rhodes applications as well

4. JavaScript and CSS Obfuscation http://edgedocs.rhomobile.com/rhodes/configuration

5. Reloadable application bundle for Android: http://docs.rhomobile.com/rhodes/build#upgrade-application

6. All Barcode issues should be fixed.

7. Windows Mobile and CE: Full screen mode supported for Hybrid and Native apps

8. Support iPhone printing via AirPrint

Usage Notes:
1. Clean build for Android take a considerable amount of time and is only required to be done if any extensions or capabilities have been added/removed.

Known Issues:
1. Camera does not work on Motorola Windows Mobile and Windows CE devices
2. On MK3000 system-api-sample gets stuck at the loading page.
3. On ET1, there is no provision to check/tell the if engine being used by the application is Motorola Webkit engine or native engine. Motorola RhoElements Splash screen does not come up with barcode-moto, webkit-browser extensions and Motorola capability.
4. Tabs does not works with Motorola Webkit
5. Inline Signature Capture does not work on Android ET1
6. Chinese characters does not displayed on Android Motorola WebKit

Supported devices:
MC75ax,MC959X,MC919X,MC319X,ES400,MC65,MK4000,MC55A0,MC55N0,MC21XX,MC759X,MC557X,VC609X,Nimbus ET1
